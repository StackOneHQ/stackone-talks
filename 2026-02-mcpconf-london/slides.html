Part 2: The Break

Three problems that emerge at scale

Problem 1

Context Explosion

Every tool definition goes into the context window.
1,000 tools × ~200 tokens each = 200,000+ tokens just for tool descriptions.

Watch the latency spike

🎬 LIVE DEMO: Simple request with 1,000 tools loaded

The agent spends more time reading tool descriptions than doing work.

Problem 2

Ambiguity & Wrong Routing

Multiple providers have similar tools.
Multiple accounts for the same provider (prod, staging, EU, US).

Problem 3

Safety

Every MCP server you connect is a trust boundary.
Every tool description is a prompt injection surface.

Part 3: The Fix

Solutions you can apply today

Fix 1

Tool Discovery (Meta Tools)

Don't load 1,000 tool definitions.
Give the agent 2-3 meta tools to search and execute.

From 1,000 tools to 3

// Instead of 1,000 tool definitions:
tools: [
  search_tools,    // Find tools by intent
  execute_tool,    // Run a discovered tool
  list_accounts    // See available accounts
]

// Agent asks: "Find tools for employee time off"
// Returns: 3 relevant tools from the right provider

Intent-based discovery

🎬 LIVE DEMO: Agent searches for tools, picks the right one, right account

Context: 200k tokens → 2k tokens. Accuracy: guessing → precise routing.

Fix 2

Code Mode / Execute

What if the action isn't a pre-defined tool?
Let the agent write and execute code against the API.

Menu vs. Kitchen

// Pre-defined tools = the menu
// Code mode = having a chef

execute({
  intent: "Get all employees hired in the last 30 days
           who haven't completed onboarding",
  account: "workday-prod"
})

// Agent writes the query, executes it, returns results
