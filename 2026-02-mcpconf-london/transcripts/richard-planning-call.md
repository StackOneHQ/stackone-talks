# Planning Call — MCP Conference Demo

**Source:** [Granola Notes](https://notes.granola.ai/t/633c0c9c-c20f-4265-96cf-f3e770a38d03)

## Key Decisions

### Talk Title
"Killing Your Agent with 10,000 Tools" (or similar provocative variation)

### Format
- 35-minute slot at MCP Conference attached to Container Days
- Target: Enterprise developers (banks, insurance, large corps)
- Goal: Technical credibility + subtle StackOne positioning

### Content Approach
- Demo agent with 1,000+ tools via StackOne MCP servers
- Focus on problems that emerge: context optimization, safety, tool discovery
- Avoid direct StackOne pitch — let the product demonstrate capabilities naturally
- Include enterprise systems (Workday, Salesforce) vs startup tools (Slack, Notion)

## Technical Demo Components

### Core Demonstration
- Start with coding agent using Claude/AI SDK
- Add 20+ MCP servers with different account IDs
- Show progressive breakdown as tool count increases
- Highlight context optimization challenges

### Safety Demonstration (if feasible)
- Show third-party MCP server compromising agent
- Contrast with StackOne safety features preventing attacks
- Target enterprise security concerns

### Backup Plan
- Pre-recorded real-time video for fallback/booth use

## Audience & Positioning

- Expected attendees: 200+ employee companies, enterprise developers
- Companies include: Adidas, American Express, HSBC, Airbus, AstraZeneca
- 90% enterprise vs 10% startups

### Messaging Strategy
- Establish technical credibility through problem-solving expertise
- Demonstrate StackOne capabilities without sales pitch
- Address enterprise-specific challenges (legacy systems, security)
- Closing: QR code for free access request with manual provisioning disclaimer
